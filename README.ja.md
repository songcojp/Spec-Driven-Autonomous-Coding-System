# Spec-Driven Autonomous Coding System

SpecDrive AutoBuild は、長時間実行可能で監査可能なソフトウェアデリバリーを支援する、Spec 駆動の自律型コーディングシステムです。

言語： [English](README.md) | [中文](README.zh-CN.md) | 日本語

---

## 概要

SpecDrive AutoBuild は、ソフトウェアチーム向けの長時間実行可能な自律型コーディングシステムです。構造化された Spec によってプロダクト目標と受け入れ基準を管理し、Skill によって再利用可能な開発手法を定義し、Subagent によってコンテキストを分離してタスクを実行し、Codex Runner によってコード変更、テスト、修正、PR 作成を行います。さらに、内部タスク状態マシンによってタスクの進行、レビュー、復旧、納品を管理し、Kanban はその状態を表示します。

一言で言うと：

> AI が単にコードを書くのではなく、制御可能で復旧可能、かつ監査可能な開発プロセスの中でコードを継続的に納品するための仕組みです。


## 主な機能

* 自然言語の要求から構造化された Feature Spec を生成する。
* PR、RP、PRD、EARS 形式の要求を、追跡可能な Feature Spec に分解する。
* Spec から技術計画、タスクグラフ、受け入れ基準、リスクルールを生成する。
* 大きな作業を、境界が明確でコンテキストが分離された Subagent Run に分割する。
* 各 Subagent に必要最小限のコンテキストだけを渡す。
* Codex Runner によってコーディング、テスト、修正、PR 作成を実行する。
* タスクが完了、失敗、ブロック、またはレビュー待ちかを自動判定する。
* 長時間実行、リトライ、再開、復旧、納品監査をサポートする。

## MVP スコープ

MVP では以下を対象とします：

* Spec Protocol とプロジェクト作成。
* Skill の登録、実行、バージョン管理。
* Subagent Runtime と Agent Run Contract。
* Context Broker と Evidence Pack。
* Codex Runner 連携。
* 内部タスク状態マシン、Kanban 状態表示、状態検査、失敗復旧。
* Review Center、PR 作成、納品レポート。

## ロードマップ

プロジェクトは以下のマイルストーンに沿って発展させます：

### M1：単一プロジェクトの自律開発

単一のソフトウェアプロジェクトに対する完全な自律デリバリーループを構築します。システムは、プロジェクト初期化、Spec 管理、作業選択、タスク生成、Codex CLI 実行、エビデンス収集、失敗復旧、レビュー記録、PR、デリバリーレポートの作成を支援します。

### M2：複数 CLI ランタイム対応

Runner 抽象を導入し、初期の Codex Runner 以外のコーディング CLI にも対応できるようにします。ランタイムは、対応する CLI 間でコマンド実行、サンドボックスポリシー、コンテキスト注入、エビデンス収集、エラー処理、結果レポートを統一します。

### M3：開発プロセスの強化

自律作業を支える開発ライフサイクルを改善します。対象には、より詳細なテスト計画、段階的な検証、品質ゲート、リリースエビデンス、デプロイ準備、環境チェック、デプロイ実行、ロールバック指針、明確な人間の承認ポイントが含まれます。

### M4：複数プロジェクト対応

システムを単一プロジェクトからプロジェクトポートフォリオへ拡張します。プラットフォームは、プロジェクト登録、プロジェクト別メモリ分離、個別の Spec とボード、共有 Skill ガバナンス、クロスプロジェクトスケジューリング、ポートフォリオレベルの可視化、複数のアクティブなデリバリーストリームの安全な協調を支援します。
